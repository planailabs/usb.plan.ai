# Architecture

Two Astro projects in one pnpm workspace; merged into one `dist/`.

## Layout

```
/
├── package.json              # main app + orchestration scripts
├── astro.config.mjs          # main app (static, base at root)
├── pnpm-workspace.yaml       # packages: [., starlight, packages/*]
├── src/pages/                # main app routes
├── public/                   # main app static
│   └── docs/                 # GENERATED by build:docs — gitignored
├── dist/                     # final merged output (gitignored)
│   └── docs/
├── packages/
│   └── remark-base-links/    # @plan/remark-base-links — shared link-prefix plugin (+ node:test)
└── starlight/
    ├── package.json          # name: "usb-docs"
    ├── astro.config.mjs      # base: '/docs', server.port: 4322
    ├── src/content/docs/     # all docs content (.md/.mdx)
    ├── src/content.config.ts
    └── public/               # docs static assets (favicon, etc.)
```

## Build flow

```
build:docs:  pnpm --filter usb-docs build  →  starlight/dist/
             cp -R starlight/dist  →  public/docs/
build:app:   astro build (root)  →  dist/  (sweeps public/* including docs/)
```

Sequential — main consumes `public/docs/`. Don't parallelize without restructuring (see dev-build skill).

## Key config

| Where | What | Why |
|---|---|---|
| `starlight/astro.config.mjs` | `base: '/docs'` | Prefixes Starlight-generated links/assets |
| `starlight/astro.config.mjs` | `server.port: 4322` | Avoid main app collision |
| `starlight/astro.config.mjs` | **No `outDir` override** | Outside-project outDir breaks Astro image cache (intermediate written to `outDir/_astro/`, read from `<project>/.astro/_astro/`) |
| `pnpm-workspace.yaml` | `packages: [., starlight, packages/*]` | Single `pnpm install` from root; `packages/*` holds shared libs like `@plan/remark-base-links` |
| `pnpm-workspace.yaml` | `allowBuilds: { sharp, esbuild }` | Required for postinstall |
| `.gitignore` | `/public/docs/` | Build artifact |

## Link prefixing

Starlight does **not** auto-prefix absolute-path markdown links with `base`. We fix this via the `remarkBaseLinks()` remark plugin from the `@plan/remark-base-links` workspace package (`packages/remark-base-links/`), imported into `starlight/astro.config.mjs` and wired into `markdown.remarkPlugins`. The plugin is extracted (not inline) so reused docs setups share one implementation and one test suite — see the `astro-starlight-setup` skill. It walks every body markdown link node — if `url` starts with `/` and doesn't already start with `/docs/`, it prepends `/docs`. Covers regular `.md`/`.mdx` doc pages; idempotent.

What it does NOT cover:
- **Frontmatter** is not part of the markdown AST. Splash `hero.actions[].link:` and any custom URL frontmatter field must be written as full `/docs/...` paths.
- **Splash template body markdown** (`template: splash` pages) — Starlight renders these through a different pipeline that bypasses `markdown.remarkPlugins`. Write their body links + hero actions as full `/docs/...` paths.

Validate via `curl -s <url> | grep -oE 'href="[^"]+"' | sort -u`.

## Other gotchas

- Astro `redirects` config: *source* gets `base` applied (so `/` resolves to `/docs/`), but *destination* does **not** — write the full path (`/docs/foo/`, not `/foo/`). Current: `/` → `/docs/start-here/welcome/`. **File-based routes take precedence over `redirects:`** — an `index.md` at the docs root would silence the redirect.
- `pnpm install` from inside `starlight/` walks up to workspace root anyway — always install from repo root.
- `build:docs` uses `rm -rf` + `cp -R` — Unix-only.

## Adding content

- Docs: `.md`/`.mdx` in `starlight/src/content/docs/`. Path = URL (relative to `/docs/`). New top-level folders need a `sidebar` entry in `starlight/astro.config.mjs`.
- Docs static assets: `starlight/public/`. Path = URL (relative to `/docs/`).
- Docs images: `starlight/src/assets/`, ref relatively in MDX.
- Main app pages: `src/pages/`.
- Main app static: `public/` — **never `public/docs/`**.
